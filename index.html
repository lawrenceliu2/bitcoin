What is Bitcoin?

First decentralized digital currency

  • Operated by a multiple entities all around the world, known as miners
  • Not tangible, only exists online (ie. riot points, mesos, arcane dust)
  • No real value, driven by the people and the system
First implementation of a cryptocurrency
  • Digital currency that uses encryption to regulate creation/transaction instead of a central authority
  • First theorized in 1998, developed in 2009

How do you get Bitcoins?

  • Buying them with US Dollars, Euros, etc. through a bitcoin exchange
  • As payment for goods or services
  • Exchange them with someone near you
  • Through mining

What are the advantages of Bitcoin?

  • Can be transferred from person to person -> No need to go through a bank -> Fees are much lower
  • Less risky for merchants (protection from fraud)
  • Payment is as easy as sending a text
  • Security and control (no unwanted changes can be forced and no personal information used in transactions, backup and encryption available)
  • Transparent and neutral

What are the disadvantages of Bitcoin?

  • Very volatile due to amount of Bitcoins in circulation
  • Still a developing project, many features are incomplete and new tools, features, and services are always in development
  • They are not widely accepted and their value fluctuates
  • Risk of technical flaws
  • Price of electricity is generally higher than value of bitcoins mined

The Bitcoin Blockchain

Each transaction in bitcoin is its own separate entity, containing the source of the sender’s bitcoins, recipient, amount, and signature, as well as a few other pieces of data. Several transactions are collected into what is known as a block, and are attached to the current transaction history (the block chain) as a node, storing the hashed index of the block before it. Because the blocks in the block chain are only capable of referencing one previous block, the order of the transactions is safeguarded. The block chain is shared between all bitcoin users, and is not encrypted, which allows any user to view the currency’s entire transaction history if they really wanted to. Transactions in a block are considered as having occurred at the same time. Transactions not yet placed in a block are known as unconfirmed. One of the most important roles of the blockchain is preventing double spending and other such fraud.

Bitcoin Mining

Mining is a process of both verifying transactions and distributing bitcoins. In the process of mining, unconfirmed transactions are gathered into a block and then suggested to the network as one possible configuration. The structure of a block is used for some complex mathematical calculations, and is verified if it passes these checks; more on this later. Bitcoin miners try to input random numerical values in order to get the output of the calculations below a certain threshold. The first person to be met with success broadcasts the block they formed to the rest of the network, and it is added to the chain. A block is mined every ~10 minutes. If two blocks are added to the chain at the same time, machines work with whatever they received first. At the next solution, all members of the network are standardized to the longest chain, keeping everyone on the same page. When a block is solved, the user is given a reward in bitcoins (free money!), incentivizing the process. The reward halves each year, so eventually it will hit 0 BTC (est. by 2140). In that case, miners will receive transaction fees for including those transactions in their blocks to continue incentive.

Proof of Work

A CPU Cost Function is computed by using trial-and-error, which is costly in terms of power consumption. Bitcoin uses a type of CPU cost function called HashCash. Each HashCash header is separated into different fields, each field containing different data relevant to verifying a valid proof of work.

Here is an example HashCash header: 1:20:1303030600:adam@cypherspace.org::McMybZIhxKXu57jd:ckvi
The first field indicates the HashCash format version, in this case it is 1. The second field indicates the number of partial pre-image bits, also known ast he number of zeroes that precede the hashed code. The third field is a timestamp which indicates when the message was sent. The fourth field indicates the resource data string being transmitted, for example, an IP address or email address. The fifth field indicates an extension, which includes miscellaneous data being transmitted. The sixth field indicates a random string of characters encoded in base-64 format. The seventh field indicates a binary counter (up to 2^20) encoded in base-64 format.
In order to prepare a valid proof of work, a sender must prepare a header and append a counter value initialized to a random number. It then computes a 160-bit SHA-1 hash of this header. If the first 20 bits of the hash are zeroes, this is a valid header. If not, the sender increments the counter and tries again. It would take around 2^20 tries before a valid header is found. With reasonable estimates, a valid header would take about 1 second to find.
To verify whetehr or not a header is valid, the recipient calculates the 160-bit SHA-1 hash of the entire string, if the first 20 bits are all zero, this is a valid header. Next, the recipient checks the date in the header, if the time stamp is more than two days old, the header is invalid. This two-day window compensates for differences in time zones and latency. Then, the recipient checks whether the email address in the header matches any email addresses registered by the recipient. If a match is not found, the header is invalid. Finally, the recipient inserts the hash string into a database, if the string is already in the database, then the string is invalid. If the header passes all of these tests, it will be considered a valid hash string.
